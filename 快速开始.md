# Revisit v1.0：快速开始

这是一个可以在 Mac 上本地运行的 Web App + Chrome 插件。第一版不需要账号、API Key 或付费服务。

## 第一步：启动 Web App

1. 解压 `revisit-v1.0.zip`。
2. 打开 Mac 的「终端」。
3. 输入 `cd `，把解压后的 `revisit-v1.0` 文件夹拖进终端窗口，然后按回车。
4. 依次运行：

```bash
npm install
npm run dev
```

5. 在 Chrome 打开：

```text
http://localhost:5173
```

只要你在使用本地版，终端中的 `npm run dev` 就需要保持运行。

## 第二步：安装 Chrome 插件

1. Chrome 地址栏打开：`chrome://extensions`
2. 打开右上角「开发者模式」。
3. 点击「加载已解压的扩展程序」。
4. 选择：`revisit-v1.0/apps/extension`
5. 在 Chrome 工具栏中固定 `Revisit`。

## 第三步：开始使用

### 保存当前网页

打开一篇文章或一条具体的 X 帖子后，点击 Chrome 工具栏中的 Revisit 插件。你可以先修改 `Article name`，再选择阅读状态、补一句备注，然后点击 `Save page`。

### 快速保存局部内容

在网页中点击右键，可以选择：

- `Save page to Revisit`
- `Save selected text to Revisit`
- `Save link to Revisit`
- `Save image to Revisit`

### 阅读与批注

1. 在 Library 点击内容卡片进入 Reader。标题右侧的铅笔按钮可以修改已经保存的文章名称。
2. 选中一个段落中的文字。
3. 页面底部会出现高亮工具条。
4. 可以直接高亮，也可以添加第一条批注。
5. 页面级笔记与高亮批注统一显示在 **My notes**；界面不区分两种 note，但点击与高亮关联的 note 时，会自动定位并强调对应原文。
6. 删除 note 时会出现一次确认弹窗，确认后立即删除。
7. 阅读文章时，将鼠标移到图片上，会显示删除按钮。点击后确认一次，即可从当前文章移除图片；若没有其他文章使用同一份去重图片，本地文件也会立即清理。
8. 所有高亮仍可在 Highlights 页面集中回看。

## 数据保存在哪里

首次启动后，本地数据会生成在：

```text
apps/server/data/reading-inbox.json
```

需要备份时，复制这个文件即可。保存过文章配图后，还需要一起备份：

```text
apps/server/data/media
apps/server/data/media-index.json（可选）
```

## 当前版本的边界

- 数据只保存在当前 Mac，没有云同步。
- X 的具体帖子页会尝试识别文字、外部网站、视频或图片。X 页面结构可能变化；识别失败时仍会保存原始链接和可编辑标题。Pinterest、登录后页面或其他复杂动态网页仍可能只能保存链接。
- 微信文章会尝试按原文顺序保存文字与图片；单次最多本地缓存 60 张图片。普通图片会转成 WebP、最长边压到 1600 px、质量 80；超长图使用 2400 px、质量 82。相同图片内容只保存一份。Reader 中每张图片都支持手动删除；图片会立刻从当前文章移除，当最后一个引用也被删除时，对应本地文件会自动清理。文章删除或重新保存后，不再被任何内容引用的本地图片同样会自动清理。下载失败时保留原图片地址作为兜底。
- 同一篇文章的高亮目前限制在单个段落内。
- 暂时没有手机收集、AI 总结、OCR 和自动关联。

## 语言显示规则

- 英文原文保留英文标题和正文，不自动翻译。
- 中文原文保留中文标题和正文，不自动翻译。
- Web App 的 `Add link` 弹窗和 Chrome 插件都可以自定义 `Article name`；留空时使用自动识别标题或域名。
- Library 列表和 Reader 右侧的 Summary 内容统一使用中文，方便快速扫读。
- 导航、按钮、筛选等产品界面继续使用英文。
- 高亮引用始终保留原文语言。
- 当前本地版尚未接入任何付费 AI 摘要服务；Summary 可以手动编辑，后续确认服务与费用后再接入自动生成。


## 更新已有本地版本

如果你已经保存过内容，升级时先备份旧项目中的 `apps/server/data/reading-inbox.json`。如果旧版本已经存在 `apps/server/data/media`，也一并复制到新版本相同位置。旧图片不会在启动时强制重压缩；重新保存对应文章后，才会替换为压缩后的 WebP，并清理不再引用的旧文件。

## 微信文章正文没有保存时

新版插件会直接读取 Chrome 当前已经显示的公众号正文。更新后请：

1. 打开 `chrome://extensions`，对 Revisit 插件点击“重新加载”。
2. 刷新微信公众号文章页面，等待正文完整显示。
3. 再打开插件。保存按钮上方应显示 `Readable body detected`。
4. 插件状态会同时显示正文段落数和图片数。
5. 即使这篇文章之前已经是 link-only 或只有文字，再保存一次也会补全原条目的正文和配图，不会新建重复内容。

## X Smart Capture（无需模型）

在一条具体的 X 帖子页面点击插件后，保存前会显示检测结果：

- `Text post`：把帖子正文作为主要内容。
- `Website share`：解析 `t.co`，尝试获取目标网站标题、简介、封面和可阅读正文。Library 与 Reader 优先展示目标网站，原 X 文案作为分享语境保留。
- `Video post`：保存视频封面和跳转入口，不下载完整视频。YouTube、Vimeo、Loom、TikTok、Bilibili 等外部视频链接也会按视频资源展示。
- `Image post`：保存当前帖子中的图片，并使用现有 WebP 压缩、尺寸限制和去重策略。

插件中的标题如果被你手动修改，会始终保留你的名称；没有修改时，系统优先使用目标网站或视频的标题。

这一版不使用模型，因此不会自动生成视频字幕、中文总结、图片说明或 Thread 总结。
